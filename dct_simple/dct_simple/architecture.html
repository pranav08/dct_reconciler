The shape is the architecture. The LLM is sandwiched between deterministic Python on both sides. It is asked to do one job — parse text into a Pydantic model — and nothing else.
Two safety layers sit on top of the sandwich:
• Grounding — every numeric value the LLM emits must trace as a substring of the source text (deterministic post-check). Catches semantic hallucinations that constrained decoding can't.
• HITL — when validator says ERROR, or judge says human_review, or judge confidence < 0.4, the record routes to a human review queue. ~2% of records. The pipeline is fast because most records skip the LLM, and safe because the unsure ones always reach a human.

The Lossy Parser

Every extracted field is Optional. The LLM is allowed to say "I don't know." Honest extraction, never fabrication.

class RawObservation(BaseModel):
    vs_code:       VitalSignCode | None = None
    value_numeric: float         | None = None

    measured_at:   datetime      | None = None
Maps to → schemas/observation.py
3

Compiled AI

Use the LLM to extract facts. Use Python for everything else — lookups, math, rules, hashes. The 1000× cost reduction lives here.

def to_si(value, unit, vs_code):
    # Pure Python. Microseconds. $0 per call.
    if vs_code == BODY_TEMP and unit == FAHRENHEIT:
        return ((value - 32) * 5.0/9.0, CELSIUS)
    return (value, unit)
Maps to → Layer 2 (computation/engine.py)
4

Deterministic Validator

Each rule is a named Python function with a stable rule_id. A regulator can re-run the validator and get bit-identical results.

@rule("VS-003", WARN, "value_numeric", ...)
def check_hr_range(obs, report):
    if obs.vs_code == HEART_RATE:
        if not (40 <= obs.value_numeric <= 200):
            report.add(# named finding)
Maps to → Layer 3 (validator/rules.py)
5

The Conditional Judge

The judge LLM never sees clean records. It fires only on WARN/ERROR findings — empirically ~15% of input.

def judge(obs, val_report):
    if not val_report.needs_judge:
        return None  # 85% exit at $0
    # Haiku via tool_use — output also constrained
    return self.client.messages.create(...)

Why so cheap?

  • Layer 1 uses a local model (Outlines + Mistral-7B). On-prem GPU = ~$0 marginal cost per call.
  • Layers 2 + 3 are pure Python. Microseconds, not seconds. $0 per call.
  • Layer 4 is conditional. Skips ~85% of records entirely. The 15% that fire cost ~$0.001 each via Haiku.
The 1000× isn't marketing. It falls directly out of the architecture: the LLM is asked to do exactly one job — parse text into a Pydantic model. Everything else is Python.

The Two Constrained-Decoding Libraries

This pipeline features both. Pick the one that fits your deployment.

Outlines (primary)

How it works: token-level FSM constraint. Compiles a Pydantic model into a finite-state machine. At every generation step, masks logits so only schema-conformant tokens can be sampled.

Guarantee: the model structurally cannot emit invalid output.

Best for: local models · strict guarantees · high throughput · regulated industries

generator = outlines.generate.json(model, RawObservation)
raw_obs   = generator(prompt)

Instructor (alternative)

How it works: response-level retry. Asks the model for JSON, validates against the Pydantic schema, retries on failure (up to max_retries).

Guarantee: the model can emit invalid output, but it gets caught and re-asked.

Best for: cloud APIs (OpenAI / Anthropic) · prototypes · 5-minute integration

client = instructor.from_anthropic(anthropic.Anthropic())
raw_obs = client.messages.create(
    model="claude-haiku-4-5",
    response_model=RawObservation,
    max_retries=3,
)

Anthropic tool_use (Layer 4)

How it works: JSON schema enforced on tool arguments. The model must call the named tool with arguments that validate against the schema.

Where used: the conditional judge LLM. The judge cannot return prose; it must structure its output.

response = client.messages.create(
    tools=[{"name":"submit_judgement", "input_schema": ...}],
    tool_choice={"type":"tool","name":"submit_judgement"},
)
The pattern: wherever an LLM speaks, force structure. Token-level (Outlines), response-level (Instructor), or API-level (tool_use). Choose your weapon — but never let a healthcare LLM emit free prose into a downstream system.

ALCOA++, 21 CFR Part 11, Grounding & HITL

Compliance falls out of the design. Every LLM-touched record carries its full provenance forward; ambiguous records always reach a human.

Grounding — extract what's in the source, never invent

The LLM has no RAG corpus, no retrieval, no external context. The source text is its only evidence. After extraction, a deterministic check verifies every emitted numeric value can be located as a substring of the source:

def verify_grounding(raw):
    src = raw.source_record.lower()
    if raw.value_numeric is not None:
        candidates = {f"{v:g}", f"{v:.0f}", f"{v:.1f}", f"{v:.2f}"}
        if not any(c in src for c in candidates):
            return ["value not in source — possible hallucination"]
    if raw.subject_id and raw.subject_id.lower() not in src:
        return ["subject_id not in source"]

Outlines prevents structural hallucinations (invalid enums); this catches semantic ones (plausible values that aren't in the source).

HITL — when the machine isn't sure, it stops

Three triggers route a record out of automation entirely:

def should_route_to_hitl(val_report, judge_report, threshold=0.4):
    if val_report.error_count > 0:
        return HITLReason.VALIDATOR_ERROR        # urgent
    if judge_report:
        if judge_report.suggested_action == "human_review":
            return HITLReason.JUDGE_REQUESTED
        if judge_report.plausibility_score < threshold:
            return HITLReason.JUDGE_LOW_CONFIDENCE
    if val_report.needs_judge and judge_report is None:
        return HITLReason.JUDGE_UNAVAILABLE
    return None

~85% records fully automated · ~15% reach the judge · ~2% reach a human. The reviewer opens the ReviewPacket (input vs output side-by-side) and either approves (creates an ESignature), rejects, or amends.

Tracked Corrections — Part 11 §11.10(e)

Originals are never overwritten. An Amendment is appended:

log.record_amendment(Amendment(
    source_id      = "v-0007",
    field_name     = "value_numeric",
    prev_value     = 72,           # preserved forever
    new_value      = 70,           # the correction
    reason_code    = "transcription_error",
    reason_text    = "Source CRF says 70; OCR misread.",
    actor          = "dr.smith@trial.org",
    prev_chain_hash = "<links to event being amended>",
))

Reason codes are a controlled vocabulary: transcription_error, data_entry_error, source_correction, investigator_review, monitor_query_resolution, regulatory_request, other. Free text without a valid code is rejected at the API boundary.

Side-by-Side Review Before Signing

A clinical data manager doesn't sign blindly. The audit log builds a ReviewPacket bundling raw input, canonical output, validator findings, and judge rationale — and the e-signature binds the entire packet:

packet = log.build_review_packet(
    source_id        = "v-0007",
    raw_input        = original_vendor_json,
    canonical_output = canonical_observation,
)
# Reviewer sees raw_input vs canonical_output side-by-side,
# reads validator findings + judge rationale, then signs.
sig = ESignature.sign(
    user_id  = "dr.smith@trial.org",
    record   = packet,            ← binds entire packet
    meaning  = "approved-for-submission",
)

If anyone tampers with either side of the packet after signing, sig.verify(packet) returns False.
