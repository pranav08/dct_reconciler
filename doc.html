AI Is Not Magic — It's a Lossy Parser

How I built a clinical trial data pipeline that's 170× cheaper than the naive cloud‑LLM approach

Most LLM pipelines in healthcare fail not because the model is wrong, but because it's asked to do the wrong job. The model gets handed a 50-line clinical narrative and a fuzzy mandate: "figure this out."

There's a better mental model. Stop thinking of the LLM as a worker. Start thinking of it as a lossy parser.

Concept 1 — Constrained Decoding: The LLM Cannot Hallucinate

The trick: the schema is the grammar.

generator = generate.json(model, RawObservation)
raw_obs = generator(prompt)

This ensures the model literally cannot emit invalid structure or enums.

Concept 2 — The Lossy Parser: Letting the Model Say "I Don't Know"

class RawObservation(BaseModel):
    subject_id: str | None = None
    vs_code: VitalSignCode | None = None
    value_numeric: float | None = None

Instead of forcing completion, allow blanks. This eliminates hallucination pressure.

Concept 3 — Compiled AI: Extract Once, Compute Forever

def to_si(value, unit):
    if unit == "F":
        return (value - 32) * 5 / 9
    return value

LLM generates logic once → Python executes millions of times deterministically.

Concept 4 — Deterministic Validator

@rule("VS-003")
def check_hr_range(obs):
    if obs.vs_code == "HR" and not (40 <= obs.value_numeric <= 200):
        flag_warning()

Every rule is traceable, testable, and auditable.

Concept 5 — Conditional Judge LLM

if not val_report.needs_judge:
    return None

# Only ~15% of records reach here

LLM is used only when deterministic logic is insufficient.

The Sandwich Architecture

Layer 1: Constrained extraction (LLM)
   ↓

   ↓

   ↓
   ├── Clean → done
